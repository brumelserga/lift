1) Участвуюют - лифт, менеджер лифтов, стратегия и классы проверки состояния лифта.
2) При вызове должен приехать оптимальный лифт (например, тот который ближе всего и свободен или тот который уже едет к данному этажу - определяется стратегией)
3) Приложение вызывает лифт - просит менеджер предоставить лифт, принимая параметром этаж с которого идет вызов.
Менеджер для каждого лифта с помощью стратегии получает количество баллов и возвращает лифт с максимальным количеством балов.
У стратегии есть классы проверки состояния лифта, которые возвращают определенное количество балов (или возможно просто 0 и 1) при выполненнии соответствующих условий.
При этом стратегия влияет на конечный результат путем установки разных наборов этих проверок (можно также добавить коофициенты для балов разных проверок).
